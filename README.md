# CapAndThief

우선 Google Gemini 로 만든 게임입니다.

🚓 게임 기획 및 규칙서
1. 게임 개요
장르: 탑다운 2D 아케이드 추격 액션

버전: v0.0.1

플랫폼: Web (HTML5 Canvas + JavaScript)

목표: 끊임없이 침입하는 도둑들을 체포하고 경찰서로 이송하여 최고의 점수를 기록하는 것.

2. 승리 및 패배 조건
승리: 없음 (무한 스코어링 방식).

패배 (Game Over): 도시 내 도둑의 총인원이 30명을 초과하면 도시가 함락되어 게임이 종료됩니다.

랭킹: 게임 종료 시 이름, 점수, 생존 시간을 Google Spreadsheet에 저장하여 순위를 경쟁합니다.

3. 플레이어 (경찰)
조작:

W, A, S, D: 이동

Shift: 부스트 (스태미나 소모, 약 1.8배 가속)

M: 위성 스캔 (미니맵에 적 위치 표시)

ESC: 일시정지 (조작 가이드 표시)

무게 시스템 (페널티):

도둑을 체포할 때마다 경찰차에 태우게 되며, 1명당 이동 속도가 0.15씩 감소합니다.

많이 태울수록 걷는 것보다 느려지므로 자주 경찰서로 복귀해야 합니다.

경찰서 (본부) 상호작용:

위치: 맵 정중앙 (World 1-1 구역).

진입 시 효과:

범인 이송: 호송 중인 범인을 감옥에 넣고 점수 획득 (+1명당 100점).

차량 정비: 무게 페널티 초기화 (속도 복구).

보급: 부스트 스태미나 100% 충전.

정보: 위성 스캔 1회 자동 발동.

4. 적 (도둑) AI 및 스폰
스폰(침입) 규칙:

초기 간격: 10초마다 1명씩 생성.

난이도 상승: 점수 100점당 스폰 간격이 1초씩 감소 (최소 1초까지 단축).

이동 속도:

기본 속도: 플레이 시간과 검거 횟수에 비례하여 지속적으로 상승.

도주 속도: 플레이어가 반경 200px 내로 접근 시, [기본 속도 + 1.5]의 속도로 도주.

행동 패턴:

평소에는 맵을 무작위로 배회(Roam).

플레이어가 접근하면 반대 방향으로 전력 질주(Flee).

5. 아이템 시스템
생성 규칙: 게임 시작 1분(01:00) 후부터 생성되며, 일정 간격으로 도로 위에 랜덤 드랍됩니다.

종류:

❄️ TIME FREEZE (10초):

모든 도둑이 이동 불가 상태가 됩니다. (체포는 가능)

다음 도둑 스폰 타이머가 일시 정지됩니다.

화면 테두리에 얼음 효과가 생기며 서서히 녹아내려 남은 시간을 알려줍니다.

🚧 SLOW AURA (30초):

플레이어 주변 반경 200px에 주황색 **억제장(오라)**이 생성됩니다.

오라 범위 내에 들어온 도둑은 속도가 60% 감소합니다.

6. 맵 (World Map)
구조: 3 x 3 그리드 방식의 월드 (총 9개 구역).

이동: 화면 끝으로 이동하면 인접한 구역으로 화면이 전환됩니다.

미니맵: 우측 상단에 현재 위치, 적(빨강), 아이템(하늘/주황), 경찰서(파랑)를 실시간으로 표시합니다.
